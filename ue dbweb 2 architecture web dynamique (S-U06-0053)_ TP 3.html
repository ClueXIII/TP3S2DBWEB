TP 3

Sujet no 3 - Bases du PHP

Quel serveur PHP ?

Deux solutions s'offre à vous : la machine pedago ou le serveur local (type XAMP). Les deux sont présentées dans la partie "En pratique". Durant les TP nous conseillons d'utiliser la machine pedago (en L2 cela deviendra même obligatoire), et de réserver le serveur local pour vos essais à la maison (même si pedago est utilisable même depuis l'extérieur du CERI, cf la présentation de cette solution).

Serveur PHP sur la machine pedago

La machine que vous utiliserez pour les TP DBWeb avec serveur distant est pedago.univ-avignon.fr. Les instructions complètes sont sur : https://dosi.univ-avignon.fr/faq/serveur-web-et-base-de-donn-es/etd-adm-ens-inv (ne pas lire la partie sur les Bases de données, ca sera pour le S3). En résumé, le serveur Apache lancé sur la machine pedago ira lire directement à la racine du répertoire public_html lorsque vous utiliserez l’url http://pedago.univ-avignon.fr/~uapvXXXX/ dans un navigateur (avec XXXX votre identifiant).

Donc le dossier public_html de votre espace personnel sur pedago devra contenir vos codes sources. Deux situations :

  1. Sur une machine du CERI : vous devez créer le répertoire public_html dans le répertoire "données itinérantes" de votre session de travail locale. Il sera ensuite visible automatiquement dans votre répertoire personnel sur pedago.univ-avignon.fr. 
  2. Sur votre machine perso : vous devez transférer vos fichiers entre votre machine perso et pedago à l'aide de la commande scp (par exemple "scp -r TP1 uapv1234@pedago:~/Donnees_itinerantes_depuis_serveur_pedagogique/public_html/.", avec votre numéro uapv1234 à vous, ou la version par fichier : sans le "-r"), dans un fenêtre "Terminal".

Attention : en cas de problème d'exécution lors de la requête http à pedago, il faut vérifier  les droits d’exécution sur votre répertoires public_html sur pedago et tout répertoire et tout fichier à l’intérieur de public_html, accessibles par le serveur apache, après chaque modification de fichier. Pour cela vous devez vous connecter sur le serveur pedago avec la commande "ssh pedago.univ-avignon.fr" puis ajuster les droits avec la commande "chmod -R 755 public_html". Pour accéder à la machine pedago par ssh depuis l'extérieur de l'université un VPN doit être démarré sur votre machine. Voir les instructions sur le site de la dosi pour son installation.

Conseil (pour plus tard) : pour simplifier vos accès à la machine pedago vous pouvez inscrire votre clé ssh publique. Pour cela suivez les instructions, par exemple, sur https://doc.ubuntu-fr.org/ssh, partie 3.2.

1. Prise en main de pedago

A faire :

  1. Démarrer votre VPN (si vous n'êtes pas en salle machine au CERI)
  2. Créer le fichier HTML/PHP hello.php contenant le code suivant dans un répertoire TP3 : 
    <html>
     <head></head>
     <body>
     Hello word en HTML !
     <?php echo "Hello world en PHP!"; ?>
      </body>
    </html>
  3. Transférer le fichier sur pedago
  4. Dans le navigateur (Firefox, IE, Chrome...), saisir l'adresse http:/pedago/~uapv????/hello.php pour voir le résultat.

3. Définition des tableaux PHP

Il existe plusieurs façons de définir et d'utiliser les tableaux en PHP. Les différences résident principalement dans l'utilisation de l'index : est-ce un entier ? ou une chaîne de caractères ? est-il donné explicitement ?...

Des exemples de code sont donnés ci-dessous, il est ensuite demandé d'adapter l'exemple à un cas particulier après l'avoir inséré dans le cadre d'un fichier HTML valide et de l'avoir testé et analysé au préalable. Toutes les questions suivantes peuvent être traitées dans un seul fichier.

Tableaux simples

EXEMPLE DE CODE

<?php
    // Declaration 1
    $tab_a[0] = "P";
    $tab_a[1] = "H";
    $tab_a[2] = "P";
    $tab_a[3] = 5;
    echo "<p>Tableau A = ";
    print_r($tab_a);
    echo "</p>";
	
    // Declaration 2 
    $tab_b = array("P", "H", "P", 5);
    echo "<p>Tableau B = ";
    print_r($tab_b);
    echo "</p>";
	
    // Declaration 3
    $tab_c[] = "P";
    $tab_c[] = "H";
    $tab_c[] = "P";
    $tab_c[] = 5;
    echo "<P>Tableau C = ";
    print_r($tab_c);
    echo "<p style=\"color:red\"> La case 3 du tableau C contient\n";
    echo "la valeur $tab_c[3]</p>\n"
?>

A faire :

  1. Ecrire vos nom, prénom et pseudo par le biais de 3 tableaux (un pour chaque mot) en utilisant 3 modes de déclaration différents (un par tableau).

4. Modularité et conception de site

Parmi les fonctions de PHP, la fonction include permet rendre modulaire la conception d'un page web. En effet, les différentes parties d'un pages peuvent être incluses dynamiquement. Nous allons mettre en oeuvre la technique sur un site simplifié qui pourra ressembler à ca :

A faire :

Elaborer le fichier HTML index.php qui doit être obtenu par l'insertion des 4 fichiers suivants :

  • entete.php
  • menu.php
  • contenu : page1.php ou page2.php ou page3.php...
  • pied.php

L'entête et le menu sont fixes. Par contre, le pied de page comprend la mention de la dernière date de modification du fichier courant. Pour cela, on pourra utiliser les fonctions filemtime pour récupérer l'information et date pour la mettre en forme de façon lisible. Le texte particulier à la partie de contenu (ie page1 ou page2...) est inséré entre le menu et le pied de page (et c'est page1.php par défaut). Dans l'exemple ci-dessus, on a prévu 2 pages : page Intro (celle affichée, court texte) et page Formulaire (ou on a utiliser le formulaire du TP1 tout simplement !)

Pour ce faire, la menu est obtenu à partir d'un tableau associatif :

  • L'association se fait entre le nom affiché sur la page et l'URL de la page visée. 
    par exemple : $TABPAGES = array("Page Intro"=>"page_intro.php","Page Formulaire"=>"formulaire.php", "Page Toto"=>"page_toto.php")
  • Dans index.php on teste si $_GET['page'] existe et on fait l'include sur sa valeur :
    if (isset($_GET['page'])){ 
       include($_GET['page']); 
    } else {    
       include(page_intro.php); 
    }
  • Donc dans menu.php, on doit générer 1 lien par page présente dans le tableau. En utilisant une boucle foreach sur TABPAGES en récupérant ses paires clé-valeur pour générer la liste des liens : ...<li><a href="index.php?page=page_intro.php" title="Page Intro">Page Intro</a></li>...
  • Ainsi, pour inclure de nouvelles pages au site, seul le tableau devra être modifié et un nombre quelconque de pages peuvent être facilement gerées de cette manière.

Bien sur le reste du site doit remplir vos nouveaux critères de qualité avec du bon HTML+CSS. Et par exemple w3.css comme dans l'exemple de l'image ci-dessus. Pour cela vous devez partir de la structure d'une page d'un TP précédent, et éviter de repartir de 0 à chaque fois. Sauf si vous voulez de la nouveauté, bien sur !
